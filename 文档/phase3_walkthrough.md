# 第三阶段 (Phase 3)：深耕 UI 组件与 Stitch 设计高保真还原

## 1. 这一阶段又做了什么突破（“软装优化”）？
如果你觉得上一个版本还不够像，在这次的改造后，我们的首页就彻底有了《传家宝日志》的正式版雏形了。
这次我们完全废弃了 Android 系统那种方方正正、刻板的自带顶部导航条，纯手工实现了一种全新的**“双层毛玻璃 (Glassmorphism Custom Header)”**，同时打造了完全对接四大模块的底部菜单。

---

## 2. 新功能与文件改动清单

| 我们改动的文件 | 对你有什么影响/可以学到什么？ |
|---|---|
| **`/gradle/libs.versions.toml`**<br>及<br>**`feature:home/build.gradle.kts`** | **扩充弹药库**。除了基础图标外，因为我们想要设计稿里那个酷炫闪烁星星的 `Discover` 图标，所以我们引入了高级图标拓展包 `material-icons-extended`。 |
| **`FoodAtlasBottomBar.kt`**<br>(全新组件文件) | **底部导航基座**。我们写了一个 `NavigationBar`，装载了 `Home`, `Discover`, `History`, `Profile`，并且你点它时所浮现的指示器圆块颜色，精确地调成了你在设计里指定的 `Sage Green`（鼠尾草绿，柔和的高级绿外发光效果）。 |
| **`HomeScreen.kt`**<br>(页面大修) | 1. **定制了漂浮在顶部的问候语**。我们将原先自带的单行生硬标题，重构成了一个 `Box`。这样上层的毛玻璃 (`alpha=0.8, blur=20.dp`) 就拥有了充足的高级感渐变高度，而且能容纳下中文的双行长问候。 <br> 2. **关联底部菜单**。我们把刚写好的 BottomBar 填入了 `Scaffold(bottomBar = {...})` 配置口。 |

## 3. 毛玻璃层级原理解析

在这个阶段，很多初学者写列表滚动时，最容易出现“划出来的文字和顶上的标题挤在一起变的一团黑”。
你可以看看我们在 `HomeScreen.kt` 里是如何解决这个专业级 UI 问题的，它的实现结构图如下：

```mermaid
graph TD
    A[Scaffold 手机壳体] --> B{Z轴空间划分}
    B -->|底层界面 (zIndex = 0)| C[LazyColumn 瀑布流]
    B -->|漂浮顶层 (zIndex = 1)| D[毛玻璃多行头部]
    
    C -->|里面内容是| C1(菜品列表)
    
    D --> E(透明度80%的背景墙)
    D --> F(高像素高斯模糊特效 blur=20dp)
    D --> G(中文题词)
    
    C -. "当底层文字往上滚时" .->|会滑进顶层的玻璃后面| F
```

由于顶层的 `blur` 是“吃掉下方正在滚过的颜色”，一旦我们精美的大图（比如酸红色的酸辣汤图片）往上划过这个玻璃区域，状态栏也会泛出一丝非常微弱的红光，这就是行业里常说的**环境色彩流失感 (Ambient Bleed)**。

## 4. 下一步：见实效
你现在不用去研究深奥的代码逻辑也可以检验成果了：

1. **重新同步项目**：不管你改了啥大库，遇到新东西，务必先在 Android Studio 点击 **Sync** 大象图标。
2. **手机部署测试**：再次点击运行。
3. 当你用拇指在主屏幕上**往下滑动**那些菜谱时，你会深刻地发现：这不仅是一个完全变成了顺眼中文界面的 App：
   - 最下巴的位置，出现了四个带着设计规范特定色系的优雅菜单。
   - 最牛的是！你的照片被慢慢拉到顶部的时候，并不是突然消失，而是**融化**在了顶部的玻璃状态栏下。

如果测试结果跟你设计稿基本达成 1:1，请指示下一步（是否想给它们加上点击动画，或者搭建实际的数据处理/业务引擎层）！
